Перед початком
Перевірте права на виконання скрипта init-db.sh:
chmod +x init-db.sh

Запустіть проект за допомогою Docker Compose:
docker-compose up --build

Це створить та запустить всі необхідні контейнери, а також виконає ініціалізацію бази даних.

Опис
Цей проект використовує Docker Compose для управління кількома сервісами, включаючи Nuxt.js додаток, Node.js сервер та PostgreSQL базу даних.

Nuxt.js додаток буде доступний за адресою http://localhost:3000.
Node.js сервер буде доступний на порту 8080.
PostgreSQL база даних буде доступна на порту 5432.
Файли та папки
docker-compose.yml - основний файл конфігурації Docker Compose.
init-db.sh - скрипт для ініціалізації бази даних.
backup.dump - дамп бази даних для відновлення.
portfolio/ - директорія з вихідним кодом Nuxt.js додатку.
server-portfolio/ - директорія з вихідним кодом Node.js сервера.

Примітки
Переконайтесь, що у вас встановлені всі необхідні залежності для роботи Docker та Docker Compose.
Якщо у вас виникають проблеми з запуском контейнерів, перевірте логи контейнерів для додаткової інформації.
