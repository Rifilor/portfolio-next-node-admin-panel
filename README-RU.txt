Перед началом
Проверьте права на выполнение скрипта init-db.sh:
chmod +x init-db.sh

Запустите проект с помощью Docker Compose:
docker-compose up --build

Это создаст и запустит все необходимые контейнеры, а также выполнит инициализацию базы данных.

Описание
Этот проект использует Docker Compose для управления несколькими сервисами, включая Nuxt.js приложение, Node.js сервер и PostgreSQL базу данных.

Nuxt.js приложение будет доступно по адресу http://localhost:3000.
Node.js сервер будет доступен по порту 8080.
PostgreSQL база данных будет доступна по порту 5432.
Файлы и папки
docker-compose.yml - основной файл конфигурации Docker Compose.
init-db.sh - скрипт для инициализации базы данных.
backup.dump - дамп базы данных для восстановления.
portfolio/ - директория с исходным кодом Nuxt.js приложения.
server-portfolio/ - директория с исходным кодом Node.js сервера.

Примечания
Убедитесь, что у вас установлены все необходимые зависимости для работы Docker и Docker Compose.
Если у вас возникают проблемы с запуском контейнеров, проверьте логи контейнеров для дополнительной информации.
